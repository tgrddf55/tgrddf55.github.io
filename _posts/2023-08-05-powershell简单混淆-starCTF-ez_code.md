# 第一部分

第一部分是预处理操作，为了给后面两个部分解混淆，先给几个变量赋值0~9。其中${-''}变量在过程中是变化的，最后这个变量被赋值为iex这个字符串。

具体来说，${$%} 之前的代码让各个变量与0~9分别对应，然后``` ${$%} = "[" + "$(@{  })"[${(}  ] + "$(@{  })"[  "${!;*}${``*%}"  ] + "$(@{  }  )  "[  "${*@ }${]}"  ] + "$?"[${!;*}  ] + "]"``` 是将```"[CHar]"```这个字符串赋给${$%} 。其中[]操作是对字符串中某个字符的索引操作，$(@{  })变量是空哈希表，如果查看这个变量的值的话是空的，但是在两边加上双引号的话就会变成字符串类型的```”System.Collections.Hashtable“```，之后的索引操作都是对这个字符串进行操作的。$?是一个特殊变量，在这里两边加上双引号之后就变成了字符串类型的```”True"```，在这里是为了索引到“r”这个字符的。powershell里对字符串拼接的操作更加随意，举个例子 ，["${!;**}${#.}"]中${!;*}值是1，${#.}值是6，那么整体含义就是[16]。 

接下来是``` { ${-``} = "".("$(  @{}  )  "[  "${!;*}${ ]}"  ] + "$(@{})  "[  "${!;*}${#.}"] + "$(  @{  }  )"[${]}] + "$(@{}  )"[${ ]}] + "$? # 第二部分和第三部分

第二部分是将一个很长很长的字符赋值给${@*}变量。通过上面的分析，将各个变量进行直接替换，然后进行字符串拼接即可得到真正的字符串。将这个变量iex一下然后echo一下，就可以打印出一个py文件。通过逆向py文件就可以得到最后的flag。 echo iex(${@ * })   

第三部分将所有的变量进行如上替换后就会得到接下来运行的代码。

```powershell
Function _/==/=__=_{
    [CmdletBinding()] param(
        [Parameter(Position = 0)]
        [String]
        $param1
    )
    $result = [Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($param1))
    return $result
}

Function _\/_\_={
    [CmdletBinding()] param(
        [Parameter(Position = 0)]
        [String]
        $param1
    )

    $param1 = _/==/=__=_ -param1 $param1
    $result  = [Text.Encoding]::UTF8.GetString([Convert]::FromBase64String($param1))
    $result | out-null
}

_\/_\_= ("S21OMFpudG9hR2hmY0hkemFGOXBjMTlsWVhONVgzSnBaMmgwUDMwPQ==")
echo "Do you konw PWSH?"
```

仔细阅读一下，第一个函数是解密一次base64字符串，第二个函数调用了一次第一个函数，相当于解密两次base64字符串,但是将解密的结果丢掉不显示。向第二个函数传入一个base64字符串，最后打印"Do you konw PWSH?"。

# 经验积累

powershell脚本中有许多与其他语言不同的地方和特性，因此这里进行一下经验总结。

1.  |是管道，大概意思是将前面的作为参数传给后面的指令进行执行。
2. 几乎任何东西加双引号之后都可以变成字符串类型，至于变成字符串后是什么则由变换之前的变量类型决定。具体详见第一部分。
3. ${@{  }}是空哈希表，反正可以加双引号。
4. 字符串可以直接替换拼接，非常灵活，这一点即使是python也无法做到。举例：["${!;**}${#.}"]意思是[16]。
5. iex命令，执行参数的代码。
6.  ISE中的操作：F9切换断点。F11单步执行。
7. echo   ${-``}.GetType().Name 可以打印出这个变量的变量类型
8.  Get-Variable 可以显示当前的变量及其值。
