我最开始的想法：直接在elements修改check_ip函数，一直返回true，从而达到绕过验证的目的，验证一直失败。

wp讲了四种方法。

  1. 禁用javascript
  2. console中输入 check_ip=function(){}
  3. hackbar发包
  4. burpsuite抓包发包

下面是gpt告诉我为什么我的不行，但是第二中方法行

为什么 check_ip = function(){} 能生效？

这是因为你在 Console 中执行了全局变量重新赋值：

  • check_ip = function(){} 这条语句是在 全局作用域 window 对象上重新定义了一个新的函数。
  • HTML 中 <form onsubmit="return check_ip()"> 会在提交时从 window.check_ip 上取这个函数来执行。

所以你等于是用一个“空函数”替换掉了原先定义的函数。只要这个函数没有 return false，表单就会默认提交。

❌ 为什么你直接修改 <script> 中的 check_ip() 函数体却没效果？

如果你在 DevTools 的 Sources 面板中直接双击 check_ip() 所在的 <script> 内容去改（或者右键“Edit as HTML”改 <script> 标签），这不会生效的原因是：

📌 1. <script> 的 JS 是一次性执行的

  • 页面加载时，浏览器会把 <script> 中的 JavaScript 执行一遍；
  • 然后就不会再执行或“重新解释”那段代码了；
  • 所以你在 Elements 里改 <script> 的内容，不会再次执行它，也不会更新函数。

📌 2. 函数已经“绑定”到 form 的 onsubmit 上了

  • 在 <form onsubmit="return check_ip()"> 中，浏览器其实会把这段 inline 代码翻译为一个函数，并与当前的 check_ip 函数绑定；
  • 如果你后来去改了 check_ip() 的函数体，但 onsubmit 事件仍然指向旧的版本（不是你修改后的）；
  • 所以你看到“改了函数体，但结果不变”的情况。

✅ 为什么重新赋值 check_ip = function(){} 就能生效？

因为这是 直接把全局的 check_ip 指针指向了一个新函数，而 onsubmit 在触发时会去读取 window.check_ip()，于是就执行了你新定义的空函数。

大概意思是说，最开始的时候check_ip就已经绑定上onsubmmit了，之后我怎么修改都没用，绑定的还是老的check_ip，但是console上我是把全局变量改掉了，因此能成功。
